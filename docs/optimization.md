# 最適化設計（量子アニーリング・割当最適化）

最適化結果は候補であり最終決定ではない。確定は必ず支援員・管理者が行う（人間中心の意思決定支援）。

## 定式化の手順

1. 業務目的を定義する
2. バイナリ変数を定義する
3. 制約を定義する
4. 目的関数を定義する
5. QUBO へ変換する
6. 必要に応じて Ising 形式へ変換する（H = Σ h_i Z_i + Σ J_ij Z_i Z_j + C、ライブラリによる自動変換を使用）
7. シミュレータまたは実機で実行する

## 基本変数

利用者 i を実習先 j の期間 t に割り当てる場合に 1 となるバイナリ変数: `x(i,j,t) ∈ {0,1}`

## ハード制約（必須）

- 同一利用者を同一期間に複数企業へ割り当てない
- 実習先の受け入れ人数を超えない
- 利用者が参加できない日には割り当てない
- 対応不可能な配慮事項を持つ企業には割り当てない
- 実習期間が重複しない
- 確定済み実習と競合しない

対応不可能な組み合わせは、可能な限り最初から変数を生成しない。

## ソフト制約（できるだけ満たす）

希望職種との一致 / スキルとの一致 / 配慮事項との適合 / 通勤負担の軽減 / 未経験職種への挑戦 / 特定企業への偏り防止 / 利用者間の実習機会の公平性 / 過去の満足度 / 過去の疲労度 / 継続勤務希望 / 苦手作業の連続回避

## QUBO の構成要素と重み

適合度 / 希望一致度 / スキル一致度 / 公平性 / 移動負担 / ローテーション価値 / 制約違反ペナルティ

支援員画面では係数を直接入力させず、業務用語（希望を重視 / スキルを重視 / 公平性を重視 / 新しい経験を重視 / 移動負担を重視）で調整させる。

## ソルバー

共通インターフェース経由で切り替え可能にする（2026-06-12 確定。依存は D-Wave Ocean SDK のみ使用）:

| キー | 実装 | 用途 |
|---|---|---|
| `sqa` | 自作 SQA（経路積分モンテカルロ。src/sqa.py、numpy のみ） | **通常運用の既定**（2026-06-11 決定）。量子アニーリングのシミュレーション。seed 固定で再現可能。変数 500 個まで |
| `sa` | dwave-samplers SimulatedAnnealingSampler | 古典手法。大規模問題（変数 500 個超）のフォールバック |
| `exact` | dimod ExactSolver（全探索） | テストの正解基準（変数 ~20 個まで） |
| `dwave` | D-Wave 実機（Ocean SDK） | 研究検証 / 最終成果の実証 |

- OpenJij・OR-Tools は不採用（SQA は依存を増やさず自作 / 古典厳密解の比較機能は不要と判断）
- 実機利用は管理者のみ。実行回数・変数数・タイムアウト・利用料金に制限を設ける

## 実機利用時の記録

量子サービス名 / ソルバー名 / 論理変数数 / 物理量子ビット数 / 埋め込み情報 / Chain Strength / Chain Break Fraction / Num Reads / 実行時間 / 待ち時間 / 最良エネルギー / 解の出現回数 / スケーリング倍率 / 利用料金

## 結果の再検証（必須）

ソルバーの結果はそのまま採用しない。別の検証処理で以下を確認する:

- 二重割当がない
- 定員超過がない
- 禁止組み合わせがない
- 日程重複がない
- 必須配慮を満たしている
- 確定済み割当と競合しない

制約違反が存在する候補は確定できないようにする。

## 説明可能性

エネルギー値ではなく、業務上理解できる提案理由を表示する:
希望職種と一致 / 必要スキルを満たす / 必要な配慮に対応可能 / 未経験職種を体験できる / 通勤負担が少ない / 過去の満足度が高い / 公平性を考慮した結果

## 実行状態

`PENDING → RUNNING → SUCCEEDED / FAILED / CANCELLED`

- 最適化実行中も通常機能は利用可能とする
- 最適化失敗時も支援員は手動で割当を作成できる
