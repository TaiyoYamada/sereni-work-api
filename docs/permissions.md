# 権限・認証認可・監査

## 利用者区分（ロール）

### 管理者（admin）
職員アカウントの作成・停止 / 利用者情報の登録・編集 / 実習先情報の登録・編集 / 全利用者の日報確認 / 実習割当の作成・修正・確定 / 自動提案の実行 / データ出力 / 監査ログ確認 / 教材管理

### 支援員（staff）
利用者情報の閲覧 / 担当利用者情報の編集 / 日報確認・コメント / 支援員評価入力 / 自動提案の実行 / 割当候補の確認・修正 / 実習スケジュール確認 / 利用者への通知

### 閲覧者（viewer）
利用者情報・日報・実習予定・実習先情報の閲覧のみ。
編集・削除・割当変更・データ出力は不可。

### 利用者（iOS アプリ）
自分の実習予定・日報・進捗・教材・通知のみ。他利用者のデータへはアクセス不可。
アカウントは支援員・管理者が発行する（自由登録なし）。

## 認証

Supabase Auth を使用: メールアドレス認証 / パスワード認証 / メール確認 / パスワード再設定 / セッション管理 / JWT 発行

## 認可（Hono API 側で確認）

1. JWT が有効である
2. アカウントが停止されていない
3. ロールが操作に対応している
4. 対象リソースへのアクセス権がある

フロントエンドでボタンを非表示にするだけでは認可としない。

## セキュリティ方針

- HTTPS / JWT 検証 / RBAC / Supabase RLS（補助的）
- 入力値検証、SQL インジェクション・XSS・CSRF 防止
- Rate Limit、ブルートフォース対策
- 秘密情報は環境変数または Secret 管理へ
- 個人情報をログへ出力しない / 機微情報を Push 通知へ含めない
- Storage は原則非公開。ファイル取得時に認可確認
- CSV 出力権限は管理者へ限定

## 監査ログ

対象操作: ログイン / ログアウト / 利用者情報閲覧・編集 / 日報閲覧・コメント・修正 / 実習割当作成・変更・確定 / 最適化実行・候補採用 / データ出力 / 権限変更 / アカウント停止

保存項目: 操作者 / 操作日時 / 操作内容 / 対象データ種別 / 対象 ID / 変更前 / 変更後 / IP アドレス / Trace ID
